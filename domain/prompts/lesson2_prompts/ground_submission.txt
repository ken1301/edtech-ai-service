<judging_rules>
- Match the student's reasoning to the approach in <approach_list> whose method it most closely follows; set matched_approach_id to that approach's index (0-based), or null if it matches none.
- CORRECT: reasoning soundly executes a valid approach with no significant gap.
- WEAK: reasoning reaches/approaches the goal but via an inefficient, fragile, or case-specific path; set matched_weakness to the key weakness.
- INCORRECT: the reasoning contains a genuine error in method or logic.
- NOT_AN_ANSWER: the submission is not an attempt to answer (blank, off-topic, "I don't know").
- If <open_approach> is true, judge the approach on its own merits even if it is not in <approach_list>.
</judging_rules>

<output_contract>
- approach_verdict: one of [CORRECT, WEAK, INCORRECT, NOT_AN_ANSWER]
- matched_approach_id: integer index into approach_list, or null
- matched_weakness: short string naming the weakness, or null
- judge_confidence: float 0..1
- explanation: one or two sentences of internal rationale (not shown to the student)
</output_contract>
